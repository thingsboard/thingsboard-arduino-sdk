135// Use the StreamUtils header internally for enabling the usage of an additonal library as a fallback, as long as the header exists,
136// to allwo to directly serialize a json message that is sent to the cloud, if the size of that message would be bigger than the internal buffer size of the client.
137// Allows sending much bigger messages than would otherwise be possible, and without the need to increase stack or heap requirements, but at the cost of increased send times.
138// See https://arduinojson.org/v6/how-to/use-arduinojson-with-pubsubclient/#serializing-a-json-document-into-an-mqtt-message for the main difference in the underlying code.
139// Option can only be enabled when using Arduino, because this feature relies on Arduino as it improves the underlying data streams to directly write the data into the MQTT Client,
140// but writing each byte one by one, would be too slow, therefore the ArduinoStreamUtils (https://github.com/bblanchon/ArduinoStreamUtils) library is used to buffer those calls into bigger packets.
141// This allows sending data that is very big without requiring to allocate that much memory, because it is sent in smaller packets.
142// To support this feature, however the IMQTT_Client interface implementation, needs to additionally override the Print interface, because that is required by the wrapper class BufferingPrint.
143# ifndef THINGSBOARD_ENABLE_STREAM_UTILS
144# ifdef __has_include
145# if __has_include(<StreamUtils.h>)
146# define THINGSBOARD_ENABLE_STREAM_UTILS 1
147# else
148# define THINGSBOARD_ENABLE_STREAM_UTILS 0
149# endif
150# else
151# define THINGSBOARD_ENABLE_STREAM_UTILS 0
