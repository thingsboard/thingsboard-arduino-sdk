211 // Vectors or array (depends on wheter if THINGSBOARD_ENABLE_DYNAMIC is set to 1 or 0), hold copy of the actual passed data, this is to ensure they stay valid,
212 // even if the user only temporarily created the object before the method was called.
213 // This can be done because all Callback methods mostly consists of pointers to actual object so copying them
214 // does not require a huge memory overhead and is acceptable especially in comparsion to possible problems that could
215 // arise if references were used and the end user does not take care to ensure the Callbacks live on for the entirety
216 // of its usage, which will lead to dangling references and undefined behaviour.
217 // Therefore copy-by-value has been choosen as for this specific use case it is more advantageous,
218 // especially because at most we copy internal vectors or array, that will only ever contain a few pointers
219#if THINGSBOARD_ENABLE_DYNAMIC
220 Vector<RPC_Callback> m_rpc_callbacks = {}; // Server side RPC callbacks vector
221#else
222 Array<RPC_Callback, MaxSubscriptions> m_rpc_callbacks = {}; // Server side RPC callbacks array
